group-by-column-aggregate

(group-by-column-aggregate column-name reducer ds-seq {:keys [skip-finalize?]})(group-by-column-aggregate column-name reducer ds-seq)

Perform a group by over a sequence of datasets where the reducer is handed each index and the context is stored in a map. The reduction in this case is unordered so your indexes will arrive out of order. The index-reductions’s prepare-batch method will be called once with each dataset before iterating over that dataset’s items.

The return value is a parallel java stream made from the entrySet of the hash map. It has an efficient further reduction to an array or you can process the stream yourself.

Options:

  • :skip-finalize? - If skip-finalize? is true then the return value simply is the reduction map containing the non-finalized reducer data. It will be up the caller to call .finalize on the reducer for each java.util.Map$Entry value returned. If finalize is true, then a parallel stream of tuples of k,v are returned.
